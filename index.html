For developers and builders

Yale Research Match is designed as a student-built layer on top of existing Yale research listings. We publish our normalized lab index as a simple JSON feed so other Yale developers can build new tools, visualizations, and integrations on top of the same data.

Public API endpoints

GET /api/labs_tagged.json           full lab index (JSON)
GET /api/labs?q=aging&dept=Chemistry  filtered search
GET /api/labs/<id>                 single lab with notes
GET /api/status                   health + index size

Core schema (labs_tagged.json)

Identity

id, name, pi_name, pi_title, department, school, lab_url, contact_email

Discovery

primary_topics, secondary_topics, tags, methods, skills_preferred, skills_optional

Fit signals

working_style, undergrad_friendly, openings_status, openings_notes, min_commitment_weeks, recommended_years

Annotations

student_notes[ ] with id, text, category, status (approved only shown), submitted_at, moderated_at

Provenance

source_flags: yura_listed, ysm_listed, dept_listed. match_metadata: topic_tokens, skills_tokens, last_indexed

Adding departments

scrape_yale_labs.py covers Chemistry, MB&B, MCDB, EEB, BME, Physics, Earth Sciences, YSPH, Statistics, and YURA. Add a new entry to the SOURCES list with the department URL and a parser key. Run the script to update labs_raw.json, then run tag_labs.py to regenerate labs_tagged.json.
